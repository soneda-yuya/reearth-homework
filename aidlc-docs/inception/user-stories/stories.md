# ユーザーストーリー — overseas-safety-map

## 構成方針

- **ペルソナ**: 登録ユーザー（1種類、[personas.md](./personas.md) 参照）
- **ブレイクダウン方式**: User Journey-Based（初回オンボーディング → 継続利用 → 通知受信 の流れ）
- **記述形式**: Classic（`As a <persona>, I want <capability>, so that <benefit>`）を基本、状況依存が強いものは **Job Story** 併用
- **受け入れ基準**: Given / When / Then（BDD 風）
- **粒度**: 中粒度（1ストーリー = 2〜3日、受け入れ基準 5〜10 個）
- **優先度**: `MVP` / `Post-MVP` の 2 区分
- **トレーサビリティ**: 各ストーリー末尾に `関連要件` として FR-ID を明記 ＋ 末尾にマトリクス

横断関心事（出典表記・通知 ON/OFF・オフライン挙動・認証エラー体験・ジオコーディングフォールバック UX・アクセシビリティ）は **単独ストーリー化せず、関連する機能ストーリーの受け入れ基準** に折り込む。

---

## ジャーニー概観

```
[初回起動] → US-01 オンボーディング / 新規登録
               ↓
[再訪]     → US-02 ログイン維持・再認証
               ↓
[日常利用] ┬→ US-03 地図で概観
           ├→ US-04 一覧で新着確認
           ├→ US-05 詳細を読む（出典表記含む）
           ├→ US-06 条件で絞り込む
           ├→ US-07 現在地周辺を確認
           └→ US-13 犯罪マップ（ヒートマップ＋カロプレス）
[継続関心] ┬→ US-08 お気に入り登録
           └→ US-09 通知設定
[通知受信] → US-10 プッシュ通知 → 詳細へ
[情報確認] → US-11 出典・利用規約ページ
[例外挙動] → US-12 オフライン・エラー時の挙動
```

## US-01: 初回オンボーディングと新規登録 `[MVP]`

**Classic**: As a 海外渡航予定の登録ユーザー, I want 初回起動時に利用規約と出典表記に同意したうえでアカウントを作成したい, so that 安心して本アプリを使い始められる.

### 受け入れ基準
- **Given** アプリを初回起動した **When** スプラッシュ完了 **Then** ログイン／新規登録画面が表示される
- **Given** 未ログイン状態 **When** 任意の画面に遷移しようとする **Then** 自動的にログイン画面へリダイレクトされる
- **Given** ログイン画面 **When** ユーザーが「新規登録」を選ぶ **Then** メールアドレス・パスワード入力フォームと、利用規約・外務省オープンデータの出典表記リンクが表示される
- **Given** 新規登録フォーム **When** 有効な入力で送信する **Then** Firebase Authentication にアカウントが作成され、Firestore に空のプロファイル（お気に入り国: 空、通知設定: OFF）が作成される
- **Given** 新規登録完了 **When** ログインに遷移 **Then** 地図画面（ホーム）が表示される
- **Given** パスワード条件不足など入力エラー **When** 送信 **Then** エラーメッセージが画面内に表示され、ユーザーは再入力できる
- **Given** 既に登録済みのメールで新規登録を試みた **When** 送信 **Then** 「既に登録されています」メッセージが表示されログイン画面へ誘導される

**関連要件**: FR-APP-05, FR-APP-07, FR-APP-04, NFR-SEC-02, NFR-SEC-04, NFR-LIC-01

## US-02: 既存ユーザーのログイン維持と再認証 `[MVP]`

**Classic**: As a 登録ユーザー, I want 2 回目以降の起動で自動的にログイン状態を維持したい、明示的にログアウトもできるようにしたい, so that 毎回ログイン作業をせずに即座に情報を閲覧できる.

### 受け入れ基準
- **Given** 前回ログインしたまま終了 **When** アプリを再起動する **Then** ログイン画面をスキップして地図画面が表示される
- **Given** Firebase トークンの有効期限切れ **When** BFF への API 呼び出しが 401 を返す **Then** アプリは無音で ID Token を再取得し、ユーザーを妨げずにリトライする
- **Given** 再取得にも失敗 **When** エラーが発生 **Then** ログイン画面に戻され、「再度ログインしてください」メッセージが表示される
- **Given** 設定メニューの「ログアウト」 **When** タップする **Then** Firebase セッションが破棄され、ログイン画面に遷移する
- **Given** ログアウト済み **When** アプリを再起動 **Then** ログイン画面が表示される

**関連要件**: FR-APP-05, FR-APP-06, NFR-SEC-02, NFR-SEC-03

## US-03: 地図で安全情報を概観する `[MVP]`

**Classic**: As a 登録ユーザー, I want ログイン後にすぐ地図を開いて、世界中の安全事象をピンで俯瞰したい, so that 関心のある地域の状況を直感的に把握できる.

### 受け入れ基準
- **Given** ログイン完了 **When** 地図画面に遷移 **Then** OSM タイルが描画され、CMS 上の安全情報がピンまたはクラスタで表示される
- **Given** ピン密集時 **When** ズームレベルが一定以下 **Then** クラスタ表示になり、件数が数字で表示される
- **Given** 地図上のピン **When** タップする **Then** タイトル・発信日・国名のポップアップが表示され、「詳細を見る」ボタンで US-05 詳細画面へ遷移する
- **Given** ジオコーディングに失敗して国セントロイドにフォールバックされた情報 **When** ピンをタップ **Then** ポップアップ内に「おおよその位置（国中心）」と注記が表示される
- **Given** 地図タイルの読み込み中 **When** 回線が遅い **Then** スケルトン／ローディング表示が出る
- **Given** アプリがオフライン **When** 地図を開く **Then** 直近表示したタイル範囲はキャッシュから表示され、「オフライン表示」バナーが出る

**関連要件**: FR-APP-02, FR-APP-03, FR-BFF-03, FR-ING-08, NFR-OPS-01

## US-04: 一覧で新着順に安全情報を確認する `[MVP]`

**Classic**: As a 登録ユーザー, I want 一覧画面で最新の安全情報を新着順に眺めたい, so that 今日どんな事象が発生したかを素早く把握できる.

### 受け入れ基準
- **Given** ホームメニュー **When** 「一覧」を選ぶ **Then** `leaveDate` の新しい順に安全情報カードが表示される
- **Given** 一覧 **When** 下までスクロール **Then** 次ページが追加ロードされる（ページング）
- **Given** カード **When** タップ **Then** 詳細画面（US-05）に遷移する
- **Given** カード表示 **When** 一覧レンダリング **Then** 各カードに `title`・国名・`leaveDate`・情報種別（infoName）が表示される
- **Given** データ取得エラー **When** ネットワーク不通 **Then** エラー表示と「再試行」ボタンが表示される
- **Given** スクリーンリーダー利用 **When** 各カードに focus **Then** 「○○の情報、○月○日、国：○○」のように読み上げが成立する

**関連要件**: FR-APP-03, FR-BFF-03, NFR-OPS-01, NFR-SEC-03

## US-05: 安全情報の詳細を読む（出典表記含む） `[MVP]`

**Classic**: As a 登録ユーザー, I want 選んだ安全情報の本文と発生場所を詳しく読みたい、ただしこれは外務省のデータであることを明示してほしい, so that 情報の出典を理解したうえで安心して読める.

### 受け入れ基準
- **Given** 地図ピン／一覧カード **When** 詳細を開く **Then** `title` / `leaveDate` / `koukanName` / `areaName`・`countryName` / `mainText` / `extractedLocation` / 緯度経度（地図ミニビュー）が表示される
- **Given** 詳細画面下部 **When** 表示される **Then** 「出典：外務省 海外安全情報オープンデータ」テキストと元記事（`infoUrl`）へのリンクが表示される
- **Given** 元記事リンク **When** タップ **Then** 外部ブラウザで外務省ページが開く
- **Given** 位置情報がフォールバック（国セントロイド）で保存されたアイテム **When** 詳細表示 **Then** 「位置情報はおおよその国の中心を示しています」の注記が表示される
- **Given** 詳細画面ヘッダー **When** 表示される **Then** 「本アプリでは位置情報を AI／ジオコーダで加工しています」の説明アイコン／ツールチップがある
- **Given** スクリーンリーダー **When** 詳細画面 focus **Then** 出典表記と本文が論理順で読み上げられる

**関連要件**: FR-APP-03, FR-APP-07, FR-ING-06, FR-ING-07, FR-ING-08, NFR-LIC-01, NFR-LIC-02

## US-06: 条件で絞り込む（検索・フィルタ） `[MVP]`

**Classic**: As a 登録ユーザー, I want 地域・国・情報種別・期間で安全情報を絞り込みたい, so that 関心のある情報にすぐ到達できる.

### 受け入れ基準
- **Given** 一覧画面または地図画面 **When** 「絞り込み」を開く **Then** 地域（areaCd）／国（countryCd）／情報種別（infoType）／期間（leaveDate 範囲）の4条件を指定できる
- **Given** 絞り込み条件を設定 **When** 「適用」 **Then** 一覧と地図の両方に条件が反映され、件数がヘッダーに表示される
- **Given** 条件を一つも指定せず適用 **When** リクエスト **Then** バリデーションエラーは出さず「全件」として扱う
- **Given** 絞り込み結果 **When** 0 件 **Then** 「該当する情報はありません」と空状態 UI を表示する
- **Given** 絞り込み条件 **When** 適用後アプリを再起動 **Then** 直前の条件が復元される（ユーザー設定として端末に保存）
- **Given** 絞り込み用プルダウン **When** 表示 **Then** 地域・国・情報種別のラベルは日本語と英語で切替可能

**関連要件**: FR-APP-03, FR-BFF-03, FR-APP-04

## US-07: 現在地周辺の安全情報を確認する `[MVP]`

**Job Story**: When 旅行中に現地に着いた直後, I want to 端末の位置情報を使って周辺の安全情報を確認したい, so I can 自分の滞在地に関わる事象を即座に把握できる.

### 受け入れ基準
- **Given** 初めて現在地機能を使う **When** ボタンをタップ **Then** OS の位置情報パーミッションダイアログが表示される
- **Given** パーミッション許可 **When** 現在地ボタンをタップ **Then** 地図が現在地にセンタリングされ、周辺（例: 半径 50km / 500km の2段階）のピンが強調表示される
- **Given** パーミッション拒否 **When** 現在地ボタンをタップ **Then** 「位置情報の利用が許可されていません。設定から変更できます」メッセージが出る
- **Given** GPS が取れない環境（屋内など） **When** 一定秒数タイムアウト **Then** 「現在地を取得できませんでした」と表示し、従来の地図ビューに戻る
- **Given** 国境近く **When** 現在地周辺を表示 **Then** 近隣国の安全情報も表示される（検索範囲は物理距離ベース）

**関連要件**: FR-APP-03（現在地）, FR-BFF-03, NFR-SEC-03

## US-08: 気になる国をお気に入り登録する `[MVP]`

**Classic**: As a 登録ユーザー, I want 気になる国をお気に入りに登録／解除したい, so that 次回以降の絞り込みや通知で優先的に扱える.

### 受け入れ基準
- **Given** 詳細画面 **When** 「お気に入りに追加」アイコンをタップ **Then** その情報の `countryCd` がユーザーの Firestore プロファイルに追加される
- **Given** 既にお気に入り登録済みの国 **When** アイコンが表示される **Then** 「追加済み」状態（塗りつぶし or ✓）で表示される
- **Given** 「お気に入り」画面 **When** 開く **Then** 登録した国の一覧と、それぞれの直近安全情報件数が表示される
- **Given** お気に入り一覧 **When** 国をタップ **Then** 該当国で絞り込んだ一覧画面に遷移する
- **Given** お気に入り解除 **When** ボタンタップ **Then** Firestore から削除され即座に UI に反映する
- **Given** Firestore 書き込みエラー **When** 発生 **Then** 「登録に失敗しました」トーストが出てロールバック表示になる

**関連要件**: FR-APP-05, NFR-SEC-04, FR-BFF-03

## US-09: 通知設定で気になる国・情報種別を選ぶ `[MVP]`

**Classic**: As a 登録ユーザー, I want 気になる国や情報種別について、新着時に通知を受け取るかを設定したい, so that 自分に関係する情報だけを受け取れる.

### 受け入れ基準
- **Given** 設定画面 **When** 「通知設定」を開く **Then** 通知のマスタースイッチ（ON/OFF）、対象国（お気に入りから選択 or 全件）、情報種別（複数選択）を指定できる
- **Given** 初回 ON 化 **When** スイッチを ON にする **Then** OS の通知パーミッションダイアログが表示され、FCM トークンが取得・Firestore に保存される
- **Given** パーミッション拒否 **When** ON にしようとする **Then** 「通知が許可されていないため受け取れません」案内と OS 設定への誘導が出る
- **Given** 通知マスターが OFF **When** 新着が配信される **Then** 当該ユーザーには送信されない
- **Given** 設定変更 **When** 保存 **Then** Firestore に即時反映され、サーバーサイド通知配信ロジックが次回配信から参照する
- **Given** 言語設定 **When** 切替 **Then** 通知設定画面のラベルも日本語／英語で切り替わる

**関連要件**: FR-APP-03（プッシュ通知）, FR-APP-05, NFR-SEC-04

## US-10: プッシュ通知を受け取り、即座に該当詳細へ遷移する `[MVP]`

**Job Story**: When 自分がフォローしている国で新しい安全情報が発生した, I want to プッシュ通知を受け取って、タップだけで詳細画面に飛びたい, so I can リアルタイムに重要情報を把握して行動に移せる.

### 受け入れ基準
- **Given** ユーザーがフォロー中の国で新着発生 **When** 取り込みパイプラインが CMS に新規アイテムを投入 **Then** 通知配信側が対象ユーザーの FCM トークンに通知を送る
- **Given** 通知を受信 **When** 通知をタップ **Then** アプリが起動（またはフォアグラウンド復帰）して該当詳細画面が直接表示される
- **Given** 未ログイン状態で通知タップ **When** 起動 **Then** ログイン後に当該詳細へ遷移する（リダイレクト保持）
- **Given** 対象アイテムが既に削除／非表示 **When** 通知タップ **Then** 「この情報は現在利用できません」フォールバック画面を表示する
- **Given** 大量の新着が同時発生 **When** 配信 **Then** ユーザー 1 人につき一定時間内にまとめて 1 件に集約するか、最大件数で制限する（通知疲れ対策）
- **Given** 通知ペイロード **When** 構築される **Then** 個人情報は含まれず、`keyCd` と title のみ

**関連要件**: FR-APP-03（プッシュ通知）, FR-ING-02〜05, NFR-SEC-01, NFR-LIC-01

- **Given** 同ページ **When** 下にスクロール **Then** 「本アプリは MOFA 原文の位置情報を LLM およびジオコーダで加工しています」旨が表示される
- **Given** 同ページ **When** 表示される **Then** CC BY 4.0 互換である旨とライセンスへのリンクが明記される
- **Given** 情報ページ **When** 日本語／英語切替 **Then** 両言語で同等の出典表記が出る

## US-13: 犯罪多発エリアを GIS で俯瞰する `[MVP]`

**Classic**: As a 登録ユーザー, I want 犯罪・事件関連の安全情報だけを、ヒートマップとカロプレスで世界地図上に可視化したい, so that 渡航先・滞在地の治安リスクを直感的に把握できる.

### 受け入れ基準
- **Given** ログイン済み **When** メニューから「犯罪マップ」を開く **Then** 世界地図が表示され、犯罪・事件関連 `infoType` で抽出された件数が **国別カロプレス**（色濃淡）で描画される
- **Given** カロプレス表示 **When** ズームインして国・都市レベルになる **Then** 自動的に **ヒートマップ**表示に切替わる（ズーム閾値は設計で決定）
- **Given** 期間フィルタ **When** 直近 7日 / 30日 / 90日 / 全期間 を切替える **Then** その期間に該当するアイテムで再集計され、地図と凡例の件数が即時反映される
- **Given** ジオコーディング失敗で国セントロイドにフォールバックされたアイテム **When** ヒートマップが描画される **Then** そのアイテムは **ヒートマップには含めず**（誤解防止）、**カロプレス集計にのみ**含める。凡例にその扱いを明記する
- **Given** カロプレスで特定国をタップ **When** 発火 **Then** US-06 の絞り込み一覧画面にその国の条件を渡して遷移する
- **Given** 凡例 **When** 画面表示 **Then** 合計件数・色スケール・「本マップは外務省領事メール（事件・犯罪関連タイプ）の発生件数を可視化したものです」旨の注記が常時表示される
- **Given** スクリーンリーダー利用 **When** 主要要素に focus **Then** 「犯罪マップ、期間○○日、合計○件、最多国○○」のサマリーが読み上げられる
- **Given** オフライン **When** 犯罪マップを開く **Then** 直近キャッシュを表示しつつオフラインバナーが出る（US-12 の共通挙動を踏襲）

**関連要件**: FR-APP-08, FR-APP-02, FR-BFF-03, FR-ING-08, NFR-LIC-01, NFR-LIC-02

## US-12: オフライン・エラー時も安心して操作できる `[MVP]`

**Job Story**: When 海外で電波が不安定な環境に居る, I want to オフラインでも最低限の情報が見える・誤解を招くエラー画面が出ない, so I can 不安を増やさずにアプリを使い続けられる.

### 受け入れ基準
- **Given** オフライン状態 **When** アプリを起動（既にログイン済み） **Then** 地図・一覧は直近のキャッシュを表示し、オフラインバナーが出る
- **Given** オフライン状態 **When** 新規データ取得アクション（絞り込み適用など） **Then** 「オフラインのためキャッシュを表示しています」トーストが出て処理は妨げない
- **Given** BFF が 5xx を返す **When** 一覧取得 **Then** 「サーバー側で問題が発生しました。しばらくしてから再試行してください」の画面内エラーが出る
- **Given** 認証エラー（401） **When** API 呼び出し **Then** US-02 のトークン再取得ロジックが作動し、失敗時はログイン画面へ
- **Given** 画面遷移時のエラー **When** 発生 **Then** クラッシュせずエラー境界が表示される（Flutter の `ErrorWidget`）
- **Given** 長時間オフライン **When** 24 時間以上古いキャッシュ **Then** 「情報が古い可能性があります」の警告が出る

### INVEST 適合チェック

| 観点 | 達成状況 |

| **Independent** | ほぼ独立。依存は US-01→US-02（認証導線）と US-08→US-09（お気に入りが通知対象の前提）のみで、並行開発可能。 |

| **Small** | 1 ストーリーあたりの受け入れ基準は 5〜7 個に収まっている。US-10 のみ通知集約要件で 6 個。 |
| **Testable** | GWT 形式で全基準が検証可能。結合テスト・ウィジェットテストに直結する（NFR-TEST-03 と整合）。 |
