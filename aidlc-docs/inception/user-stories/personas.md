# ペルソナ — overseas-safety-map

Story Generation Plan の Q1 で「登録ユーザー（B）＋認証 must」が選択されたため、本プロジェクトのペルソナは **1種類（登録ユーザー）** に統一する。運用担当・データ提供者視点は本プロジェクトのストーリー対象外（運用はインフラ作業、データ提供は間接ステークホルダー）。

---

## Persona 1: 登録ユーザー（Registered User）

### 基本プロファイル
- **呼称**: 海外出張者・留学生・海外旅行者
- **年齢層**: 20〜50代
- **利用環境**: iOS / Android スマートフォン、モバイル回線または Wi-Fi、旅行中はローミングまたは現地 SIM
- **技術リテラシー**: スマホアプリを日常的に使う一般ユーザーレベル（地図アプリ・SNS・メッセンジャーを扱える）

### 目的 / 動機
- 渡航先・滞在地の **最新の安全情報** を把握したい
- 気になる国の情報を **継続的にフォロー** したい（お気に入り登録）
- 重大な安全事象を **プッシュ通知で即座に受け取りたい**
- 「どこで何が起きたか」を **地図上で直感的に理解** したい

### ペインポイント
- 外務省 海外安全ホームページは PC 前提で、スマホでの閲覧性が低い
- 国ごとの情報がテキスト中心で、発生場所が **どこか地図上で分からない**
- 情報が多すぎて、自分に関係する国・地域の情報を絞り込みにくい
- 渡航中にリアルタイムで通知を受け取る手段が限られる

### 利用シナリオ（代表例）
1. 旅行計画時に渡航先の国を絞り込んで直近の安全事象をざっと確認する
2. 出張先に着いてから、現在地周辺のピンを地図で確認する
3. 気になる国をお気に入り登録しておき、新しい事象発生時にプッシュ通知で受け取る
4. 通知タップで詳細画面に直接遷移し、本文と外務省の元記事リンクを確認する

### 認証・セキュリティ要求
- **ログイン必須**: 未ログインではアプリを利用不可（FR-APP-05 更新版）
- 個人識別情報（Firestore のお気に入り・通知設定）は本人のみアクセス可能であること
- 認証方法は Firebase Authentication（初期はメール／パスワードを想定、設計で確定）

### 成功指標（このペルソナにとっての「使えるアプリ」）
- 起動から 3 タップ以内で地図・一覧・詳細のいずれかに辿り着ける
- 関心国を登録してから最初のプッシュ通知を受け取れる（通知配信が機能する）
- 回線が悪い環境でも地図タイル・既読データがキャッシュされ最低限の情報が見える

### 想定しない（スコープ外）
- ユーザー間のコミュニティ・口コミ機能
- 外務省以外のソースからの安全情報
- PCブラウザでの利用（Flutter Web 配信は MVP 対象外）
- オペレーション業務（パイプライン再実行、CMS スキーマ変更）
